## 一、项目概述

### 项目背景
企业级 AI 应用引擎（类似 Coze/Dify），支持快速创建对话应用和可视化工作流编排。

### 技术栈
- **后端**: NestJS 10 + TypeScript + Prisma ORM
- **前端**: Next.js 14 + React + shadcn/ui + TailwindCSS
- **数据库**: PostgreSQL 16 + pgvector
- **缓存**: Redis 7
- **AI 服务**: Ollama 本地模型 + 阿里云百炼（可选）
- **包管理**: pnpm workspace (Monorepo)

### 项目规模
- **代码量**: 约 5500+ 行（累计）
- **文件数**: 70+ 个
- **Git 提交**: 22 次
- **开发时间**: 3 天（2026-03-14 至 2026-03-16）

## 三、模块功能概述

### 3.1 应用管理模块
**功能**: 创建和管理 AI 应用，生成 API Key 用于认证。

**关键 API**:
- `POST/GET/PATCH/DELETE /api/apps` - 应用 CRUD
- `POST /api/apps/:id/regenerate-key` - 重置 API Key
- `GET /api/models` - 模型列表

**技术要点**:
- API Key 认证中间件（ApiKeyGuard）
- `@Public()` 装饰器跳过认证
- Prisma ORM 数据访问

**核心文件**:
- `apps/server/src/modules/app/app.service.ts`
- `apps/server/src/auth/api-key.guard.ts`

---

### 3.2 对话功能模块
**功能**: 与 AI 模型进行对话，支持流式和非流式响应。

**关键 API**:
- `POST /api/chat/completions` - 非流式对话
- `POST /api/chat/completions/stream` - 流式对话
- `GET /api/chat/sessions/:id/messages` - 获取消息历史

**技术要点**:
- LLM Provider 抽象层（阿里云、Ollama）
- ProviderFactory 工厂模式
- SSE 流式响应（使用 fetch + ReadableStream）
- 会话和消息存储

**核心文件**:
- `apps/server/src/modules/chat/chat.service.ts`
- `packages/providers/src/provider-factory.ts`
- `apps/web/src/lib/sse-client.ts`

---

### 3.3 工作流管理模块
**功能**: 创建和执行可视化工作流，支持多种节点类型编排。

**关键 API**:
- `POST/GET/PATCH/DELETE /api/workflows` - 工作流 CRUD
- `POST /api/workflows/:id/run` - 执行工作流
- `GET /api/workflows/:id/runs` - 执行记录

**节点类型** (5 种):
- `start` - 起始节点（设置变量）
- `llm` - LLM 调用节点
- `http` - HTTP 请求节点
- `condition` - 条件判断节点
- `end` - 结束节点
- `tool` - 工具调用节点

**技术要点**:
- DAG（有向无环图）调度
- 变量系统（多作用域：全局、节点、临时）
- 模板解析（支持 `{{ variable }}` 和 `{{ nodes.xxx.output }}`）
- 工作流执行引擎（拓扑排序）

**核心文件**:
- `packages/core/src/workflow-executor.ts`
- `packages/core/src/node-executors/` - 5 种执行器
- `apps/server/src/modules/workflow/workflow.service.ts`

---

### 3.4 工具管理模块
**功能**: 提供可扩展的工具抽象层，支持 HTTP 请求、代码执行、时间操作。

**关键 API**:
- `GET /api/tools` - 工具列表
- `GET /api/tools/:name` - 工具详情
- `POST /api/tools/:name/execute` - 执行工具

**内置工具** (3 个):
- **HTTP 工具**: 发送 HTTP/HTTPS 请求（GET/POST/PUT/DELETE）
- **代码工具**: 在安全沙箱中执行 JavaScript 代码（vm2 隔离）
- **时间工具**: 获取、格式化、转换时间（date-fns）

**技术要点**:
- Tool 接口抽象
- ToolRegistry 注册表
- ToolNodeExecutor 工作流集成
- vm2 沙箱安全隔离

**核心文件**:
- `apps/server/src/modules/tool/tool.service.ts`
- `apps/server/src/modules/tool/tool.registry.ts`
- `packages/core/src/node-executors/tool-executor.ts`

## 四、模块间联系

### 数据流向
```
应用管理 → API Key → 对话功能（认证）
    ↓
工作流管理 → ToolNodeExecutor → 工具管理（调用）
    ↓
对话功能 ← LLM Provider ← AI 服务（Ollama/阿里云）
```

### 关键集成点
1. **应用 → 对话**: API Key 认证，确保只有授权应用可调用对话 API
2. **工作流 → 工具**: ToolNodeExecutor 调用 `/api/tools/:name/execute`
3. **工作流 → 对话**: LLM 节点使用 ProviderFactory 调用 AI 服务
4. **前端 → 后端**: Axios 客户端封装，统一错误处理

## 五、开发卡点与 Bug 修复

### 5.1 主要卡点

**卡点 1: NestJS 编译后 workspace 包解析失败**
- **现象**: 编译后运行报错 `Cannot find module '@ai-engine/providers'`
- **原因**: Node.js v24 默认 ESM 与 CommonJS 配置冲突
- **解决**: 开发环境使用 `ts-node` 直接运行源码

**卡点 2: SSE 流式响应类型不匹配**
- **现象**: `@Sse()` 装饰器报错 "You must return an Observable"
- **原因**: NestJS 的 SSE 需要返回 Observable，实现返回 AsyncIterableIterator
- **解决**: 改用普通 POST + 流式 Response（`res.write`）

**卡点 3: 表单内按钮默认提交**
- **现象**: 点击节点列表跳转到工作流列表页
- **原因**: HTML 表单内 `<button>` 默认 `type="submit"`
- **解决**: 所有按钮添加 `type="button"` + `preventDefault()`

### 5.3 关键技术决策

1. **使用 ts-node 运行开发服务器**
   - 避免编译后 workspace 包解析问题
   - 命令：`npx ts-node --project tsconfig.json -r tsconfig-paths/register src/main.ts`

2. **所有表单内按钮添加 `type="button"`**
   - 防止意外触发表单提交
   - 配合 `e.preventDefault()` 和 `e.stopPropagation()`

3. **fetch 替代 EventSource 实现 SSE**
   - 支持 POST 请求（EventSource 仅支持 GET）
   - 使用 ReadableStream 处理流式响应

4. **数据结构验证前置**
   - 提交前验证 `definition.nodes` 和 `definition.edges` 是数组
   - 提供默认空数组防止后端验证失败
