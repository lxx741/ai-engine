## 四、修复的 Bug 列表

### 1. provider-factory.ts - getModelName() 逻辑错误
**问题**: `getModelName('ollama/llama2:7b')` 返回 `'7b'` 而非 `'llama2:7b'`  
**修复**: 调整正则表达式检查顺序，先检查 slash 格式  
**文件**: `packages/providers/src/provider-factory.ts`

### 2. vitest.config.ts - 路径别名错误
**问题**: `@ai-engine/providers` 导入失败  
**修复**: 修正路径为 `../../packages/providers/src`  
**文件**: `apps/server/vitest.config.ts`

### 3. $transaction mock 签名错误
**问题**: Prisma 的 `$transaction` 接收数组参数，mock 使用回调函数  
**修复**: 修改 mock 实现支持数组参数  
**文件**: `apps/server/src/modules/chat/*.spec.ts`

### 4. vi.mock hoisting 问题
**问题**: `mockProvider is not defined` 错误  
**修复**: 使用 `vi.hoisted()` 包装 vi.mock  
**文件**: `apps/server/src/modules/chat/chat.service.spec.ts`

### 5. 测试环境变量干扰
**问题**: `ALIYUN_API_KEY` 已配置导致 healthCheck 测试失败  
**修复**: 测试中临时删除环境变量  
**文件**: `packages/providers/src/__tests__/*.test.ts`

## 七、技术亮点

### 1. 全局 Mock 策略
```typescript
// apps/server/src/test/setup.ts
vi.mock('@prisma/client', () => {...})
vi.mock('@ai-engine/providers', () => {...})
vi.mock('@ai-engine/core', () => {...})
```

### 2. 直接实例化服务（避免 TestingModule 冲突）
```typescript
// 不推荐
const module = await Test.createTestingModule({...}).compile()

// 推荐
const service = new AppService(mockPrismaService)
```

### 3. vi.mock hoisting 处理
```typescript
const mockProvider = { name: 'aliyun', chatComplete: vi.fn() }

vi.hoisted(() => {
  vi.mock('@ai-engine/providers', async () => {
    return {
      getProviderFactory: vi.fn(() => ({
        getProviderForModel: vi.fn(() => mockProvider),
      })),
    }
  })
})
```

### 4. $transaction 数组参数支持
```typescript
$transaction: vi.fn(async (operations: any[]) => {
  const results = []
  for (const op of operations) {
    results.push(await op)
  }
  return results
})
```
