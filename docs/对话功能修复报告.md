# 对话功能路径报错修复报告

**修复日期**: 2026-03-16  
**问题状态**: ✅ 已修复

---

## 🔍 问题分析

### 初始现象
访问对话功能路径 `http://localhost:3001/chat/c910a00d-3bcf-4ef1-ab1a-bd68da87e17a` 时，控制台报错：
```
Failed to load resource: the server responded with a status of 500 (Internal Server Error)
- /api/chat/sessions
```

### 问题根源

经过排查发现是 **`@Query` 装饰器参数类型转换问题**：

1. **第一层问题**: `chat.controller.ts` 中的 `getConversations` 方法使用 `@Query` 装饰器获取分页参数时，没有进行类型转换
   ```typescript
   // ❌ 错误代码
   @Query('page') page: number = 1
   @Query('pageSize') pageSize: number = 20
   ```
   这导致前端传来的字符串参数（如 `"1"`）没有被转换为数字，在计算 `skip: (page - 1) * pageSize` 时得到 `NaN`

2. **第二层问题**: 尝试使用 `ParseIntPipe` 时，当参数为空时会抛出验证错误
   ```typescript
   // ❌ 也会报错
   @Query('page', ParseIntPipe) page?: number
   ```

3. **第三层问题**: 后端服务启动失败
   - NestJS 编译后无法正确解析 workspace 包（`@ai-engine/providers`）
   - Node.js v24 默认使用 ESM 但包配置为 CommonJS
   - 最终解决方案：使用 `ts-node` 直接运行 TypeScript 源码

---

## ✅ 修复方案

### 修复 1: Controller 参数处理

**文件**: `apps/server/src/modules/chat/chat.controller.ts`

**修改前**:
```typescript
async getConversations(
  @Query('page') page: number = 1,
  @Query('pageSize') pageSize: number = 20,
) {
  try {
    return await this.conversationService.findAll(page, pageSize);
  } catch (error) {
    throw new InternalServerErrorException('获取会话列表失败');
  }
}
```

**修改后**:
```typescript
async getConversations(
  @Query('page') page?: string,
  @Query('pageSize') pageSize?: string,
) {
  try {
    const pageNum = page ? parseInt(page, 10) : 1;
    const pageSizeNum = pageSize ? parseInt(pageSize, 10) : 20;
    return await this.conversationService.findAll(pageNum, pageSizeNum);
  } catch (error) {
    console.error('Get conversations error:', error);
    throw new InternalServerErrorException('获取会话列表失败', error.message);
  }
}
```

**改动说明**:
- 将参数类型改为 `string` 并设为可选
- 使用 `parseInt()` 手动转换，提供默认值
- 添加详细的错误日志输出

### 修复 2: 改进错误处理

在 controller 的 catch 块中添加错误详情输出，方便调试：
```typescript
catch (error) {
  console.error('Get conversations error:', error);
  throw new InternalServerErrorException('获取会话列表失败', error.message);
}
```

### 修复 3: 服务启动方式

由于 Node.js v24 和 workspace 包的模块解析问题，使用 `ts-node` 启动开发服务器：

```bash
# 推荐开发启动命令
cd apps/server
npx ts-node --project tsconfig.json -r tsconfig-paths/register src/main.ts
```
