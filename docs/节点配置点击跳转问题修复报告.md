## 🔍 问题分析

### 初始现象
在节点配置模块中，点击"节点列表"或"已添加节点"中的某个节点时，会直接跳转到工作流列表页（`http://localhost:3001/workflows`），无法正常编辑节点配置。

### 根本原因

经过深入分析发现 **多个问题共同导致**：

1. **按钮缺少 `type="button"` 属性**
   - 在 HTML 表单中，`<button>` 元素默认类型为 `submit`
   - 所有未指定 `type` 的按钮都会触发表单提交
   - 导致点击节点列表中的按钮时触发 `form onSubmit`

2. **事件处理不完整**
   - 部分按钮的 `onClick` 只有 `e.stopPropagation()`
   - 缺少 `e.preventDefault()` 阻止默认行为
   - 导致点击事件仍然可能触发表单提交

3. **节点移动功能未实现**
   - MoveUp/MoveDown 按钮修改了本地数组但未保存变更
   - 没有调用父组件的更新函数
   - 节点顺序修改后无法持久化

**示例**:

### 修复 3: 实现节点移动功能

**新增 `onMoveNode` 回调函数**:

**文件**: `apps/web/src/components/workflow/node-editor.tsx`

```typescript
interface NodeEditorProps {
  // ... 其他 props
  onMoveNode?: (index: number, direction: 'up' | 'down') => void
}

// 在 MoveUp/MoveDown 按钮中调用
<Button
  type="button"
  onClick={(e) => {
    e.preventDefault()
    e.stopPropagation()
    if (index > 0 && onMoveNode) {
      onMoveNode(index, 'up')
    }
  }}
>
```

**文件**: `apps/web/src/components/workflow/workflow-form.tsx`

```typescript
const handleMoveNode = (index: number, direction: 'up' | 'down') => {
  setDefinition(prev => ({
    ...prev,
    nodes: prev.nodes.map((node, i) => {
      if (direction === 'up' && i === index - 1) {
        return prev.nodes[index]
      }
      if (direction === 'down' && i === index + 1) {
        return prev.nodes[index]
      }
      if (i === index) {
        return direction === 'up' ? prev.nodes[index - 1] : prev.nodes[index + 1]
      }
      return node
    }),
  }))
}
```

## 🚀 技术要点

### 1. React 表单中的按钮类型

在 React 中，表单内的 `<button>` 元素默认行为：
```typescript
// ❌ 默认会提交表单
<button>Click me</button>

// ✅ 不会提交表单
<button type="button">Click me</button>

// ✅ 提交表单
<button type="submit">Click me</button>
```

### 2. 事件处理最佳实践

```typescript
// 完整的事件处理模式
onClick={(e) => {
  e.preventDefault()      // 阻止浏览器默认行为
  e.stopPropagation()     // 阻止事件冒泡到父元素
  // 执行实际业务逻辑
}}
```

### 3. 不可变数据更新

```typescript
// ✅ 正确：创建新对象
setDefinition(prev => ({
  ...prev,
  nodes: prev.nodes.map(...)
}))

// ❌ 错误：直接修改原对象
prev.nodes[index] = newNode
setDefinition(prev)
```

## ⚠️ 注意事项

1. **所有表单内按钮**: 必须添加 `type="button"` 除非需要提交表单
2. **事件处理顺序**: 先 `preventDefault()` 再 `stopPropagation()`
3. **状态更新**: 使用函数式更新确保获取最新状态
4. **回调函数**: 可选的回调函数使用 `?` 标记并检查是否存在
