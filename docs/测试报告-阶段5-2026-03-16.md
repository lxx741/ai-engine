## 四、问题与解决

### 问题 1: WorkflowExecutor mock hoisting
**现象**: `vi.mock('@ai-engine/core')` 在测试文件中不生效  
**根因**: vi.mock 需要 hoisting，普通代码中导入后无效  
**解决**: 使用 `vi.hoisted()` 包装，并使用外部 mock 函数引用
```typescript
const mockExecute = vi.fn()
const mockExecuteNode = vi.fn()

vi.hoisted(() => {
  vi.mock('@ai-engine/core', async () => {
    return {
      WorkflowExecutor: vi.fn().mockImplementation(() => ({
        execute: mockExecute,
        executeNode: mockExecuteNode,
      })),
    }
  })
})
```

### 问题 2: ToolRegistry 依赖注入
**现象**: ToolService 构造函数需要 ToolRegistry 实例  
**根因**: ToolRegistry 是普通类，非 NestJS 服务  
**解决**: 创建 mock 对象并直接传入构造函数
```typescript
const mockToolRegistry = {
  register: vi.fn(),
  get: vi.fn(),
  list: vi.fn(),
  execute: vi.fn(),
}

service = new ToolService(mockToolRegistry as any)
```
