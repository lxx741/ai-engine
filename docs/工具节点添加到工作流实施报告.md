## 🎯 功能演示

### 使用流程

1. **访问工作流编辑页面**
   ```
   http://localhost:3001/workflows/[id]/edit
   ```

2. **点击"节点配置"标签**

3. **添加工具节点**
   - 点击"工具"按钮（橙色）
   - 节点添加到"已添加节点"列表

4. **配置工具节点**
   - 点击"已添加节点"中的"工具节点"
   - 右侧显示配置面板
   - 选择工具类型（HTTP/代码/时间）
   - 填写节点名称
   - 填写工具参数（JSON 格式）
   - 查看工具说明

5. **连接节点**
   - 在"边配置"标签页添加边
   - 连接到其他节点

6. **保存工作流**
   - 点击"更新工作流"按钮

## 📊 支持的三种工具

### 1. HTTP 工具 (`http`)

**用途**: 发送 HTTP/HTTPS 请求

**参数示例**:
```json
{
  "url": "https://api.example.com/data",
  "method": "GET",
  "headers": {},
  "body": {}
}
```

**使用场景**:
- 获取天气数据
- 调用第三方 API
- 发送 Webhook 通知

---

### 2. 代码工具 (`code`)

**用途**: 在安全沙箱中执行 JavaScript 代码

**参数示例**:
```json
{
  "code": "const result = [1,2,3].map(x => x * 2); result"
}
```

**使用场景**:
- 数据格式转换
- 复杂计算
- 字符串处理

---

### 3. 时间工具 (`time`)

**用途**: 获取、格式化、转换时间

**参数示例**:
```json
{
  "action": "format",
  "format": "yyyy-MM-dd HH:mm:ss",
  "timezone": "Asia/Shanghai"
}
```

**使用场景**:
- 获取当前日期时间
- 格式化时间戳
- 计算到期时间

### 示例 1: 获取天气并格式化

```json
{
  "name": "天气查询工作流",
  "definition": {
    "nodes": [
      {
        "id": "start",
        "type": "start",
        "config": { "name": "开始" }
      },
      {
        "id": "tool_1",

          "name": "获取天气",

            "method": "GET",

          }
        }
      },
      {
        "id": "tool_2",
        "type": "tool",
        "config": {
          "name": "格式化天气",

        }
      },
      {
        "id": "end",
        "type": "end",
        "config": { "name": "结束" }
      }
    ],
    "edges": [
      { "source": "start", "target": "tool_1" },
      { "source": "tool_1", "target": "tool_2" },
      { "source": "tool_2", "target": "end" }
    ]
  }
```

## ⚠️ 注意事项

### 1. JSON 参数验证
- 当前实现允许输入任意 JSON
- 建议在后续版本中添加 JSON Schema 验证
- 可使用 `tool-params-form.tsx` 组件实现动态表单

### 2. 工具说明
- 当前是硬编码的说明文本
- 建议从后端动态加载工具详情
- 使用 `useTool` hook 获取工具参数定义

### 3. 错误处理
- 参数 JSON 解析失败时静默忽略
- 建议添加错误提示（红色边框 + 错误信息）
