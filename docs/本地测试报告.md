## 🔧 问题修复

### 问题：前端控制台 404 错误

**现象**: 访问 `http://localhost:3001/` 时控制台报错

**错误信息**:
```
Failed to load resource: the server responded with a status of 404 (Not Found)
- /_next/static/chunks/main-app.js
- /_next/static/chunks/app-pages-internals.js
- /_next/static/chunks/webpack.js
```

**原因**: Next.js 开发服务器进程处于暂停状态（SN），导致 HMR（热模块替换）缓存失效

**解决方案**: 重启前端开发服务器
```bash
# 杀死旧进程
kill <PID>

# 重新启动
cd apps/web
pnpm dev
```

## 📝 已知问题

根据文档记录的问题：

1. **Ollama 响应时间较长** (30-40 秒)
   - 原因：qwen3.5:9b 模型较大（6.6GB）
   - 影响：用户体验稍差
   - 建议：使用更小模型或云 API

2. **SSE 流式响应未完全实现**
   - 原因：NestJS 需要返回 Observable
   - 影响：流式对话暂时不可用

3. **工作流前端配置较复杂**
   - 原因：配置式节点编辑（非可视化）
   - 影响：用户体验一般
