## 🔍 问题分析

### 初始现象
在对话页面 `http://localhost:3001/chat/[id]` 输入"你好"并点击发送后，控制台报错：
```
Failed to load resource: the server responded with a status of 500 (Internal Server Error)
- /api/chat/completions/stream
- /api/chat/sessions/:id/messages
```

### 问题根源

经过排查发现 **3 个主要问题**：

#### 问题 1: Query 参数类型转换问题
**位置**: `chat.controller.ts` 和 `conversation.service.ts`

**现象**: 
- `@Query('limit') limit: number = 50` 导致字符串参数未转换
- Prisma 查询时 `take: limit` 得到 `NaN`

**错误**:
```
Invalid `this.prisma.message.findMany()` invocation
Argument `take` is missing.
```

#### 问题 2: SSE 流式接口实现问题
**位置**: `chat.controller.ts` 和 `sse-client.ts`

**现象**:
- 后端使用 `@Sse()` 装饰器需要返回 `Observable`
- 实际实现返回 `AsyncIterableIterator<MessageEvent>`
- 前端使用原生 `EventSource`（仅支持 GET）但后端需要 POST

**错误**:
```
You must return an Observable stream to use Server-Sent Events (SSE).
```

#### 问题 3: 参数验证问题
**位置**: `chat.controller.ts`

**现象**:
- `@Query('page')` 和 `@Query('pageSize')` 未处理空值
- 直接使用默认值导致计算错误

async getHistory(conversationId: string, limit: number = 20) {
  const safeLimit = typeof limit === 'number' && !isNaN(limit) ? limit : 20;
  const messages = await this.prisma.message.findMany({
    where: { conversationId },
    orderBy: { createdAt: 'desc' },
    take: safeLimit,
  });
  return messages.reverse();
}

### 修复 2: 流式接口改造

**后端**: `apps/server/src/modules/chat/chat.controller.ts`

```typescript
// ❌ 修改前 - 使用 @Sse() 装饰器
@Post('completions/stream')
@Sse()
streamCompletion(
  @Body() sendMessageDto: SendMessageDto,
): AsyncIterableIterator<MessageEvent> {
  // ...
}

// ✅ 修改后 - 使用普通 POST + 流式响应
@Post('completions/stream')
async streamCompletion(
  @Body() sendMessageDto: SendMessageDto,
  @Res() res: Response,
) {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');

  const stream = this.chatService.sendStream(
    sendMessageDto.message,
    sendMessageDto.conversationId,
    sendMessageDto.userId,
  );

  for await (const chunk of stream) {
    res.write(`data: ${JSON.stringify(chunk)}\n\n`);
    if (chunk.finishReason === 'stop') {
      res.write(`data: ${JSON.stringify({ type: 'done' })}\n\n`);
      break;
    }
  }

  res.end();
}
```

**前端**: `apps/web/src/lib/sse-client.ts`

```typescript
// ❌ 修改前 - 使用 EventSource（仅支持 GET）
export function createSSESource(url: string, body: any, callbacks: any) {
  const eventSource = new EventSource(url);
  // ...
}

// ✅ 修改后 - 使用 fetch + ReadableStream（支持 POST）
export function createSSESource(url: string, body: any, callbacks: any) {
  const controller = new AbortController();
  
  fetch(url, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(body),
    signal: controller.signal,
  })
    .then((response) => {
      const reader = response.body?.getReader();
      const decoder = new TextDecoder();
      let buffer = '';
      
      const processStream = async () => {
        const { done, value } = await reader.read();
        if (done) {
          callbacks.onDone?.();
          return;
        }
        
        buffer += decoder.decode(value, { stream: true });
        const lines = buffer.split('\n');
        buffer = lines.pop() || '';
        
        for (const line of lines) {
          if (line.startsWith('data: ')) {
            const data = line.slice(6);
            const parsed = JSON.parse(data);
            callbacks.onChunk?.(parsed);
          }
        }
        
        processStream();
      };
      
      processStream();
    });
    
  return {
    close: () => {
      controller.abort();
    },
  };
}
```
