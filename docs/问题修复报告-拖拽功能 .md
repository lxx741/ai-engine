## 🔍 问题诊断

### 问题 1: 后端服务未运行

**现象**:
```
Failed to load resource: net::ERR_CONNECTION_REFUSED
- http://localhost:3000/api/workflows
- http://localhost:3000/api/apps
```

**根本原因**:
- NestJS 的 `dev` 脚本使用 `nest start --watch` 会编译到 `dist/` 目录
- 编译失败或 `dist/` 目录不存在
- 服务无法正常启动

**日志**:
```
Error: Cannot find module '/Users/lxx/Desktop/ai-engine/apps/server/dist/main'
```

### 问题 2: 拖拽功能不生效

**现象**:
- 从左侧工具箱拖拽节点，画布无响应
- 拖拽时没有视觉反馈
- 释放鼠标后节点未创建

**根本原因**:
1. **缺少拖拽事件处理**: Canvas 组件没有实现 `onDragOver` 和 `onDrop` 事件
2. **Sidebar 配置不完整**: 拖拽数据格式需要与画布接收端匹配
3. **样式问题**: 缺少 `cursor-grab` 等视觉反馈样式

#### 修改文件 2: `apps/web/src/components/workflow/canvas-editor.tsx`

**修改内容**:

1. **添加节点类型定义**:
```typescript
const NODE_TYPES = [
  { type: 'start', label: '开始节点' },
  { type: 'llm', label: 'LLM 调用' },
  { type: 'http', label: 'HTTP 请求' },
  { type: 'condition', label: '条件判断' },
  { type: 'tool', label: '工具调用' },
  { type: 'end', label: '结束节点' },
];
```

2. **实现拖拽事件处理**:
```typescript
// Handle drag over (required for drop to work)
const onDragOver = useCallback((event: React.DragEvent) => {
  event.preventDefault();
  event.dataTransfer.dropEffect = 'move';
}, []);

// Handle drop - create new node from sidebar
const onDrop = useCallback(
  (event: React.DragEvent) => {
    event.preventDefault();

    const type = event.dataTransfer.getData('application/reactflow');
    if (!type) {
      console.warn('No node type in drag data');
      return;
    }

    // Calculate position relative to the React Flow container
    const reactFlowBounds = event.currentTarget.getBoundingClientRect();
    const position = {
      x: event.clientX - reactFlowBounds.left,
      y: event.clientY - reactFlowBounds.top,
    };

    // Get node label
    const nodeType = NODE_TYPES.find((t) => t.type === type);
    const label = nodeType?.label || type;

    // Create new node
    const newNode: Node = {
      id: `node_${Date.now()}`,
      type,
      position,
      data: {
        name: label,
        config: {},
      },
    };

    setNodes((nds) => [...nds, newNode]);
  },
  [setNodes]
);
```

3. **更新容器元素**:
```typescript
<div className="flex-1 h-full" onDragOver={onDragOver} onDrop={onDrop}>
  <ReactFlow ... />
</div>
```

## 📝 经验总结

### 关键学习点

1. **开发环境配置**:
   - 根据 AGENTS.md，NestJS 开发环境应使用 ts-node
   - 避免编译步骤，加快开发迭代

2. **React Flow 拖拽**:
   - 需要同时实现 `onDragOver` 和 `onDrop`
   - 拖拽数据格式必须匹配
   - 视觉反馈提升用户体验

3. **事件处理**:
   - `event.preventDefault()` 是必须的
   - 计算位置时使用 `getBoundingClientRect()`
   - 注意事件冒泡和捕获

### 踩坑记录

1. **端口占用问题**:
   - 启动服务前先释放端口
   - 使用 `lsof -ti:3000 | xargs kill -9`

2. **拖拽无效**:
   - 检查是否缺少 `onDragOver` 事件
   - 验证 `dataTransfer.setData` 和 `getData` 的键名一致

3. **样式问题**:
   - `cursor-grab` 和 `active:cursor-grabbing` 提供视觉反馈
   - `select-none` 防止拖拽时选中文本
