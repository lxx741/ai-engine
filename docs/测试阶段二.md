## 🐛 发现的问题和解决方案

### 问题 1: 向量索引 B-Tree 限制
**现象**: 插入向量时分块失败，报错 "index row size exceeds maximum"  
**原因**: PostgreSQL B-Tree 索引限制 2704 字节，mxbai-embed-large 生成 1024 维向量（4096 字节）  
**解决**: 删除 B-Tree 索引，在应用层计算余弦相似度  
**修改文件**: `fix-vector-index.js`、`knowledge-base.service.ts`  
**状态**: ✅ 已修复

### 问题 2: 前端 API 路径双重前缀
**现象**: 404 Not Found，URL 为 `/api/api/knowledge-bases`  
**原因**: baseURL 已包含 `/api`，hooks 中又添加了 `/api` 前缀  
**解决**: 修改所有 hooks 中的路径，移除 `/api` 前缀  
**修改文件**: `use-knowledge-bases.ts` (6 处)、`use-documents.ts` (4 处)  
**状态**: ✅ 已修复

### 问题 3: 搜索参数验证失败
**现象**: 400 Bad Request，提示 "topk must be a number"  
**原因**: URL 查询参数是字符串，DTO 验证需要 number 类型  
**解决**: 在 `SearchKnowledgeBaseDto` 中添加 `@Transform` 和 `@Type` 装饰器  
**修改文件**: `knowledge-base.dto.ts`  
**状态**: ✅ 已修复

### 问题 4: DTO 循环引用
**现象**: 后端启动失败，"Cannot access RAGConfigDto before initialization"  
**原因**: `CreateKnowledgeBaseDto` 在 `RAGConfigDto` 之前引用  
**解决**: 调整 DTO 类定义顺序，`RAGConfigDto` 放在前面  
**修改文件**: `knowledge-base.dto.ts`  
**状态**: ✅ 已修复

### 问题 5: PDF 解析失败
**现象**: 上传 PDF 后显示 "[PDF content - parsing not available]"  
**原因**: pdf-parse 库 ESM/CJS 兼容性问题  
**临时方案**: 返回占位符文本  
**长期方案**: 安装 pdfjs-dist 或使用命令行工具  
**修改文件**: `file-parser.ts`  
**状态**: ⚠️ 待完善

### 问题 6: OOM 内存溢出
**现象**: 后端服务崩溃，"JavaScript heap out of memory"  
**原因**: Node.js 默认内存限制（~4GB）不足  
**解决**: 设置 `NODE_OPTIONS="--max-old-space-size=8192"`  
**状态**: ✅ 已修复
