# 创建工作流失败修复报告

**修复日期**: 2026-03-16  
**问题状态**: ✅ 已修复

## 🔍 问题分析

### 初始现象
从工作流列表页面点击"新建工作流"按钮，填写基本信息后点击"创建工作流"按钮，报错：
```
创建工作流失败，请重试
```

### 根本原因

**缺少必填字段 `appId`**

在之前修复更新工作流的 400 错误时，为了符合 `UpdateWorkflowDto` 的要求，移除了提交数据中的 `appId` 字段。但是**创建工作流时 `appId` 是必需的**。

### 后端 DTO 要求

**CreateWorkflowDto** (`apps/server/src/modules/workflow/dto/workflow.dto.ts`):

```typescript
export class CreateWorkflowDto {
  @IsString()
  @IsNotEmpty()
  appId: string;        // ❌ 必填字段

  @IsString()
  @IsNotEmpty()
  name: string;         // 必填字段

  @ApiPropertyOptional({ description: '工作流描述' })
  @IsString()
  @IsOptional()
  description?: string;

  @ApiProperty({ description: '工作流定义（DAG）' })
  @IsObject()
  @IsNotEmpty()
  definition: any;      // 必填字段

  @ApiPropertyOptional({ 
    description: '工作流状态',
    enum: ['draft', 'published', 'archived'],
    default: 'draft' 
  })
  @IsEnum(['draft', 'published', 'archived'])
  @IsOptional()
  status?: string = 'draft';
}
```

### 前端提交数据问题

**修改前** (`workflow-form.tsx`):

```typescript
const handleSubmit = (e: React.FormEvent) => {
  e.preventDefault()
  
  const validDefinition = {
    nodes: Array.isArray(definition.nodes) ? definition.nodes : [],
    edges: Array.isArray(definition.edges) ? definition.edges : [],
    variables: definition.variables || {},
  }
  
  // ❌ 创建和更新都不提交 appId
  onSubmit({
    name,
    description,
    status,
    definition: validDefinition,
  })
}
```

**问题**: 
- 创建模式缺少 `appId`，导致后端验证失败
- 更新模式不应包含 `appId`，符合 DTO 要求

## ✅ 修复方案

### 修复 1: 区分创建和更新场景

**文件**: `apps/web/src/components/workflow/workflow-form.tsx`

**修改内容**:

```typescript
const handleSubmit = (e: React.FormEvent) => {
  e.preventDefault()
  
  // Validate definition structure
  const validDefinition = {
    nodes: Array.isArray(definition.nodes) ? definition.nodes : [],
    edges: Array.isArray(definition.edges) ? definition.edges : [],
    variables: definition.variables || {},
  }
  
  console.log('Submitting workflow:', {
    name,
    description,
    status,
    definition: validDefinition,
  })
  
  // ✅ 根据是否有 workflow prop 区分创建/更新模式
  const submitData: any = {
    name,
    description,
    status,
    definition: validDefinition,
  }
  
  // 如果是创建模式（没有 workflow prop），添加 appId
  if (!workflow && appId) {
    submitData.appId = appId
  }
  
  onSubmit(submitData)
}
```

**判断逻辑**:
- **创建模式**: `workflow` prop 不存在 → 添加 `appId`
- **更新模式**: `workflow` prop 存在 → 不添加 `appId`

## ⚠️ 注意事项

1. **DTO 验证**: 
   - `CreateWorkflowDto` 要求 `appId` 必填
   - `UpdateWorkflowDto` 不允许 `appId` 字段

2. **模式判断**:
   - 通过 `workflow` prop 是否存在来区分创建/更新
   - 创建模式：`workflow === undefined`
   - 更新模式：`workflow !== undefined`

3. **默认值处理**:
   - `appId` 默认选择第一个应用
   - `status` 默认为 `'draft'`
   - `definition` 默认为空节点和边数组

4. **数据验证**:
   - 提交前验证 `definition.nodes` 和 `definition.edges` 是数组
   - 提供默认空数组防止验证失败
