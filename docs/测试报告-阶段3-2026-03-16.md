## 四、问题与解决

### 问题 1: Template Helper 节点 ID 格式
**现象**: 使用 `node-1` 作为节点 ID 时测试失败  
**根因**: `parseTemplate` 使用 JavaScript 表达式求值，连字符被解析为减法  
**解决**: 使用不含连字符的节点 ID（如 `node1`）  
**影响**: 测试更准确，发现潜在设计问题

### 问题 2: TestingModule 与全局 mock 冲突
**现象**: 使用 `Test.createTestingModule` 时 `this.prisma` 为 undefined  
**根因**: 全局 setup.ts 的 vi.mock 与 TestingModule 注入冲突  
**解决**: 直接实例化服务，手动注入 mock 依赖
```typescript
// 不推荐
const module = await Test.createTestingModule({...}).compile()

// 推荐
const service = new AppService(mockPrismaService())
```

### 问题 3: generateApiKey mock hoisting
**现象**: vi.mock 不生效，API Key 仍为随机值  
**根因**: vi.mock 需要 hoisting，在普通代码中导入后无效  
**解决**: 使用 `vi.hoisted()` 包装 vi.mock
```typescript
vi.hoisted(() => {
  vi.mock('./app.service', async (importOriginal) => {
    const actual = await importOriginal()
    return { ...actual, generateApiKey: vi.fn(() => 'sk_test123456') }
  })
})
```
