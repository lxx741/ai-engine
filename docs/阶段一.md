## 项目概述

### 项目背景
企业级 AI 应用引擎（类似 Coze/Dify），支持快速创建对话应用和可视化工作流编排。

### 技术栈
- **后端**: NestJS 10 + TypeScript + Prisma ORM
- **前端**: Next.js 14 + React + shadcn/ui + TailwindCSS
- **数据库**: PostgreSQL 16 + pgvector
- **缓存**: Redis 7
- **AI 服务**: Ollama 本地模型 + 阿里云百炼（可选）
- **包管理**: pnpm workspace (Monorepo)

### 项目规模
- **代码量**: 约 3600+ 行（阶段一新增）
- **文件数**: 70+ 个文件
- **Git 提交**: 10+ 次提交
- **开发时间**: 2 天（2026-03-14 至 2026-03-15）

## 已知问题

### 问题 1：Ollama 响应时间较长

**现象**: 对话 API 响应时间约 30-40 秒  
**原因**: qwen3.5:9b 模型较大（6.6GB），本地 M4 芯片推理需要时间  
**影响**: 用户体验稍差  
**解决方案**:
- 使用更小的模型（如 qwen2.5:7b）
- 或使用云 API 服务（阿里云百炼）
- 添加加载提示

**优先级**: P1（可优化）

---

### 问题 2：SSE 流式响应未完全实现

**现象**: 流式 API 返回类型为 Observable，NestJS 需要特殊处理  
**原因**: NestJS 的 SSE 需要返回 Observable，当前实现返回 AsyncIterable  
**影响**: 流式对话功能暂时不可用  
**解决方案**:
- 修改返回类型为 Observable
- 或使用纯 HTTP 流式响应

**优先级**: P1（可优化）

---

### 问题 3：工作流前端配置较复杂

**现象**: 配置式节点编辑需要手动填写 JSON  
**原因**: 未实现可视化拖拽编排  
**影响**: 用户体验一般  
**解决方案**:
- 实现 React Flow 可视化编排
- 或提供更友好的表单配置

**优先级**: P2（可选优化）

---

### 问题 4：工具管理前端功能简单

**现象**: 只能测试工具，不能创建/编辑工具  
**原因**: 基础版只实现了查看和测试  
**影响**: 无法自定义工具  
**解决方案**:
- 添加工具创建/编辑功能
- 支持自定义工具注册

**优先级**: P2（可选优化）

---

### 问题 5：无循环支持

**现象**: 工作流仅支持 DAG（有向无环图）  
**原因**: 避免无限循环  
**影响**: 无法实现循环逻辑  
**解决方案**:
- 添加 while/for 节点类型
- 设置最大循环次数

**优先级**: P3（远期规划）
