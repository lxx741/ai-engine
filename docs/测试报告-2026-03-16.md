## 三、问题分析

### 3.1 严重问题 (P0)

#### 问题 A: Server 包 Vitest 兼容性

**影响**: 所有 Server 包测试无法运行

**根本原因**:
1. Vitest ESM 与 NestJS CommonJS 混合导致
2. `@nestjs/testing` 的依赖注入机制在 Vitest 中需要额外配置
3. Prisma Client mock 需要全局配置

**解决方案**:

**方案 1: 使用 vi.mock 全局 Mock (推荐)**
```typescript
// src/test/setup.ts
import { vi } from 'vitest'

vi.mock('@prisma/client', () => {
  return {
    PrismaClient: vi.fn(() => ({
      $connect: vi.fn(),
      $disconnect: vi.fn(),
      $transaction: vi.fn(),
      conversation: { create: vi.fn(), findUnique: vi.fn() },
      message: { create: vi.fn(), findUnique: vi.fn() },
    }))
  }
})
```

**方案 2: 回退 Jest (临时)**
```bash
# 如果时间紧迫，Server 包暂时使用 Jest
pnpm --filter @ai-engine/server add -D jest ts-jest @types/jest
```

**方案 3: 深入研究 Vitest + NestJS 配置**
- 参考：https://github.com/vitest-dev/vitest/issues/1188
- 需要配置 `experimentalVmThreads`
