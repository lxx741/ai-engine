# 工作流更新 400 错误修复报告

**修复日期**: 2026-03-16  
**问题状态**: ✅ 已修复

---

## 🔍 问题分析

### 初始现象
访问工作流编辑页面 `http://localhost:3001/workflows/[id]/edit`，点击"节点配置 → 节点列表 → 开始"配置节点后，点击"更新工作流"按钮时报错：
```
PATCH http://localhost:3000/api/workflows/c286ff6d-8055-424d-be0e-e9b80457a360 400 (Bad Request)
更新工作流失败：AxiosError: Request failed with status code 400
```

### 问题根源

经过排查发现是 **前端 `definition` 对象结构可能不规范** 导致的验证失败：

1. **`definition.nodes` 或 `definition.edges` 可能不是数组**
   - 在 `NodeEditor` 组件中，节点和边的更新操作可能导致数据结构异常
   - 某些情况下 `nodes` 或 `edges` 可能是 `undefined` 或其他类型

2. **状态同步问题**
   - `NodeEditor` 组件中的 `selectedNode` 状态与父组件的 `definition` 状态可能不同步
   - 快速修改节点配置时可能导致数据不一致

3. **缺少数据验证**
   - 提交前没有验证 `definition` 对象的结构
   - 后端 DTO 验证失败时错误信息不够详细
